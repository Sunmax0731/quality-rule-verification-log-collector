# 競合・公式基準ベンチマーク

## 比較方針

品質ルール検証ログコレクター は、競合の全機能を再実装するのではなく、自動検証だけでなく、手動検証の証跡も品質ルール単位で回収する。

| 参照先 | URL | 競合・公式標準の強み | 採用する評価基準 | 差別化 |
| --- | --- | --- | --- | --- |
| GitHub Actions | https://docs.github.com/en/actions/using-workflows/about-workflows | 自動検証の実行条件、ログ、失敗履歴をワークフローとして残せる。 | リリース、差分、検証ログ、ブランチ状態を追跡できること。 | 自動検証だけでなく、手動検証の証跡も品質ルール単位で回収する。 |
| Linear | https://linear.app/features | Issue、プロジェクト、ロードマップ、サイクルを高速に横断できる。 | 状態、優先度、履歴、判断理由を追えること。 | 自動検証だけでなく、手動検証の証跡も品質ルール単位で回収する。 |
| Jira | https://www.atlassian.com/software/jira/features/ | バックログ、ボード、レポート、承認フローを組織単位で運用できる。 | 状態、優先度、履歴、判断理由を追えること。 | 自動検証だけでなく、手動検証の証跡も品質ルール単位で回収する。 |

## 改善へ反映した点

- QCDS評価に競合比較と公式標準の確認を追加した。
- 実装だけでなく、README、導入手順、ユーザーガイド、手動テスト、リリース前資料を評価対象にした。
- 文字化けをQCDSのQuality/Satisfactionリスクとして検出する。
